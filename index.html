One-time Setup Required

Register a free Atlassian OAuth app so users can log in with their Atlassian/Google account. Takes about 10 minutes.

1
Go to developer.atlassian.com/console/myapps → Create → OAuth 2.0 integration. Name it Jira Team Dashboard.
2
Under Authorization, add this exact Callback URL:
—
3
Under Permissions, add:
• Jira API → read:jira-work, write:jira-work
• User identity API → read:me
• Enable Offline access (for token refresh)
4
Copy your Client ID and paste it below, then open this file in a text editor and replace YOUR_ATLASSIAN_CLIENT_ID permanently.

🔒 The Client ID is not a secret — it is safe to embed in this HTML file. No client secret is required (PKCE flow). Tokens are stored in session memory only and cleared when you close the browser tab.

Jira Team Dashboard

Sign in with your Atlassian account to view the Jira tickets you have reported.

This app will request access to:

✓ Read Jira issues you reported
✓ Transition your reported issues to Done
✓ Read your name and email (to identify you)
✗ Cannot read other users' data
✗ Cannot create, edit, or delete issues

🔒 You will be redirected to Atlassian's official login page. If your organisation uses Google Workspace SSO, you will see a "Continue with Google" option. Credentials are never sent to this app — only a short-lived access token is returned. Tokens are cleared when you close this tab.
